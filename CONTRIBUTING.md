# Contributing to mailshake-ruby

Thank you for your interest in contributing! Here's how to get started.

## Development Setup

1. Fork and clone the repository
2. Run `bin/setup` to install dependencies
3. Run `bundle exec rspec` to verify tests pass
4. Use `bin/console` for an interactive prompt to experiment

## Making Changes

1. Create a feature branch from `master`
2. Write your code and add tests
3. Ensure all tests pass: `bundle exec rspec`
4. Commit with a clear message describing the change
5. Open a pull request against `master`

## Guidelines

- Follow existing code style and conventions
- Add tests for any new functionality
- Update documentation (README, CHANGELOG) as needed
- Keep pull requests focused on a single change

## Reporting Bugs

Open an issue on [GitHub Issues](https://github.com/ESouza/mailshake-ruby/issues) with:

- Ruby version and gem version
- Steps to reproduce
- Expected vs actual behavior

## Security

For security vulnerabilities, please see [SECURITY.md](SECURITY.md) instead of opening a public issue.
